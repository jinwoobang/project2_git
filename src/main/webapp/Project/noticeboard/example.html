UI/UX 콘셉트기획 사용자의 목적(goals)과 행동(behaviors)을 대표하는 실제 타깃 사용자 그룹을 가상의 캐릭터(artificial person and archetypical users)로 형상화하는 목적 지향적인(goal-directed) 인터랙션 디자인의 방법은 페르소나이다 -------------------------------------------------------------------------------------------------------------------------------- UI/UX 콘셉트는 대체와 모방이 불가능하면서, 독보적인 가치를 제시할 수 있어야 하며, 선정된 프로토타입을 보완할 방향성과 사용자의 행동패턴 및 선호도에 부합하면서 비즈니스 측면에서 경쟁우위가 가능한 접근을 필수적 가져가는 전략은 VRIN전략이다 ------------------------------------------------------------------------------------------------------------------------------- 사용자 리서치 결과와 UI/UX 콘셉트에서 페르소나를 구성할 필수 요건 사용자 목적과 니즈 사용자의 행동패턴. 사용환경. 사용자의 태도. -------------------------------------------------------------------------------------------------------------------------------- 프로토타입 테스트에 해당하는 것 병행 프로토타이핑. 반복실험과 평가. 호감도 테스트. 시간인식연구. -------------------------------------------------------------------------------------------------------------------------------- Valuable, Rare, Inmitable, Nonsubstitutable 전략의 6가지 체크 리스트 A사와 B사는 경쟁관계에 있는 회사이지만 B사는 A사가 대체하지 못할 서비스나 제품을 제공하고 있다. B사의 연구실에서는 A사의 제품을 모방할 수 없다. A사는 B사보다 항상 고객에게 새로운 가치를 제공하고 있다. A사는 이번에 개발한 프로토타입으로 UX 특허를 등록한 계획을 가지고 있다. -------------------------------------------------------------------------------------------------------------------------------- 리스크나 투자 대비 최적의 성과를 창출할 수 있으면서, 사용자에게 핵심적인 기능을 최소화하여 제공할 수 있도록 정의하는 방식이다. 해당 용어는 프랭크 로빈슨(Frank Robinson)이 정의하였고, 스티브 블랭크(Steve Blank)와 에릭 리이스(Eric Ries)가 대중화시킨 제품은 최소기능제품이다 ------------------------------------------------------------------------------------------------------------------------------- 목적 및 기대효과. - 적은 기능으로 가설을 빠르게 테스트해볼 수 있다. - 기능을 학습시키는 종합적인 상황을 파악해볼 수 있다. - 불필요한 개발 비용을 줄일 수 있다. - 최단 기간 내에 초기 사용자의 피드백을 받아볼 수 있다. - 제품의 완성도를 높이는 초석이 된다. - 시장이 요구하는 제품을 적시에 출시할 가능성이 높다. 위 내용이 설명하는 것은 최소기능제품이다 ----------------------------------------------------------------------------------------------------------------------------------- 이메일을 확인하여 회원을 가입하기 기능. - 분실한 비밀호 확인하기. - 직접가입하지 않고 SNS계정으로 로그인하는 기능. 사용자용 기능 리스트이다 -------------------------------------------------------------------------------------------------------------------------------- 백엔드 기능 리스트(backend feature list)에 적합한 것은 사용자가 수행 또는 접근할 수 있는 기능을 시스템 단위에서 동작시키는 기능 목록으로 구성한 리스트이다 -------------------------------------------------------------------------------------------------------------------------------- 우리가 일상에서 발생하는 다양한 사건에서 약 20%의 원인이 80%의 결과에 영향을 미친다는 논리로서, 전체 기능 목록 중에 사용자에게 가치 있고, 중요하게 활용하는 필수적인 기능은 상위 20% 정도의 범주에 분포되어 있을 것이라고 보는 법칙은 파레토법칙이다 ---------------------------------------------------------------------------------------------------------------------------------- 사용자의 니즈와 제공하는 콘텐츠 간의 갭을 분석할 수 있으며, 갭 분석한 결과를 토대로 콘텐츠의 방향성을 결정하는 것을 콘텐츠 기획이라 한다 ---------------------------------------------------------------------------------------------------------------------------------- 파편화되고 불명확한 방대한 정보를 논리적으로 응집력 있는 그룹으로 조직화하고 분류하여 보다 더 좋은 아이디어를 발굴하거나 문제를 좀 더 이해하는 기법으로 사용하는 다이어그램은 어피니티 다이어그램이다 ---------------------------------------------------------------------------------------------------------------------------------- 사용자들이 실제로 행동하는 내적 이유를 파악하고, 공감을 형성할 수 있도록 접근하는 방법으로, 특히 사용자들이 태스크를 수행할 경우에 보이는 감정, 행동, 생각(emotions, behaviors, and beliefs) 등을 제공하고 있는 기능, 서비스, 제품(features,service offering, and product)과 비교하여 갭을 분석하는 방법(rigorous framework)으로 주로 사용하는 다이어그램은 멘탈모델 다이어그램이다 ------------------------------------------------------------------------------------------------------------------------------------ 어피니티 다이어그램의 프로세스 토픽이나 문제를 명료하게 정의한다. 팀별로 딘덱카드나 부착가능한 노트를 제공한다. 카드에 아이디어나 이슈를 기술한다. 카드를 그룹핑한 각각의 그룹에 레이블을 기술한다. ----------------------------------------------------------------------------------------------------------------------------------- 사용자가 시스템을 어떻게 사용하는지(다음에 무엇을 해야 하는지), 친숙하지 않은 시스템이나 예상치 못한 상황에서 무엇을 해야 하는지 (어떻게 시스템이 동작하는지)를 이해하는데 종종 묘사할 때 적합한 다이어그램은 멘탈모델 다이어그램이다 ------------------------------------------------------------------------------------------------------------------------------------- 콘텐츠의 방향성을 결정하는 도중 사용자의 니즈와 제공하는 서비스간에 갭 데이터 4가지 유형 사용자의 니즈와 제공하는 서비스가 일치하는 유형 사용자의 니즈는 있으나 제공하는 서비스가 없는 유형 사용자의 니즈는 있으나 제공하는 서비스가 부실한 유형 사용자의 니즈는 없는데 제공하는 서비스가 있는 유형 --------------------------------------------------------------------------------------------------------------------------------------------------- 콘텐츠 분류의 가치 및 대상 중 창조 콘텐츠는 사용자 요구사항과 개선방향을 반영하여 통합 사이트내에 새로운 콘텐츠를 제공할 목적의 콘텐츠이다. [창조 콘텐츠의 추가 및 재정립 설명] A사이트는 사용자의 개인화 콘텐츠를 추가하였다. B사이트는 사용자 맞춤 콘텐츠를 추가하였다. A사이트는 기존 고객문의 서비스에서 사용자의 선택기능을 추가하였다. B사이트는 사용자들의 참여를 독려하기 위해 사용자 참여 콘텐츠를 추가하였다. ----------------------------------------------------------------------------------------------------------------------------------------------------- 브랜드 리서치의 가치 및 대상에서 유저관점의 브랜드 리서치와 서비스관점의 브랜드 유저관점 : 맨탈모델 리서치( 사용자가 인식하고 있는 브랜드) 유저관점 : 태스크모델 리서치( 사용자가 서비스 이용을 위해 방문하고 있는 브랜드) 서비스관점 : 서비스 모델 리서치( 사용자에게 제공하고 있는 콘텐츠) 서비스관점 : 서비스 브랜드 리서치 (사용자에게 도메인으로 제공하고 있는 사이트) -------------------------------------------------------------------------------------------------------------------------------------------------------- 신규 콘텐츠를 수급하기 위해서 검토해야할 사항 외부 콘텐츠 소싱 여부 검토 - 유료 또는 무료 콘텐츠 소싱 여부. 외부 콘텐츠 소싱 여부 검토 - 자체가이드 라인 적용 수준 검토 외부 콘텐츠 소싱 여부 검토 - 비용관련 검토 외부 콘텐츠 소싱 여부 검토 - 콘텐츠 종류 검토. ----------------------------------------------------------------------------------------------------------------------------------------------------------- 콘텐츠 서비스에 대한 외주업체를 선택하게 되었다. 비용관련검토에 해당되는 것들 계약기간 검토. 유지보수 관련 비용검토. 저작권 관련비용 검토. 기타비용검토. --------------------------------------------------------------------------------------------------------------------------------------------------------
